# Data Model: AI Design Interview Wizard

## Entities

### InterviewSession
Represents a chat session between a user and the AI for a specific exam problem.

| Field | Type | Description |
|---|---|---|
| `user_id` | String (PK) | Unique identifier for the user (e.g., "demo-user") |
| `exam_id` | String (SK) | Identifier for the exam problem (e.g., "ST-2024-1") |
| `history` | List[ChatMessage] | Chronological list of messages in the session |
| `status` | String | Current status: "active", "completed" |
| `created_at` | DateTime | Session creation timestamp |
| `updated_at` | DateTime | Last activity timestamp |

### ChatMessage
A single message in the interview history.

| Field | Type | Description |
|---|---|---|
| `role` | String | "user" or "assistant" |
| `content` | String | The text content of the message |
| `timestamp` | DateTime | When the message was sent |

### DesignProposal
The structured output generated by the AI, intended to populate the Design Wizard.

| Field | Type | Description |
|---|---|---|
| `theme` | String | The central theme/thesis of the paper |
| `breakdown` | Map[String, String] | Key points for sections "ア", "イ", "ウ" |
| `structure` | List[Chapter] | Proposed chapter structure |

### Chapter
A chapter in the design structure.

| Field | Type | Description |
|---|---|---|
| `id` | Integer | Chapter number/ID |
| `title` | String | Chapter title |
| `content` | String | Brief description of chapter content |

## Relationships

- **User** (1) -- (N) **InterviewSession** (One session per exam problem)
- **InterviewSession** (1) -- (1) **Design** (The session informs the design, but they are stored separately)
