# 3.1 なぜ論文が書けないのか
午後IIの論文試験は、難しい試験だと言われている。しかし受験者はこの壁を越えなければ、合格の栄冠は得られない。午前試験で100点を取っても、最後の午後II試験で合格ラインに入らなければ全ては無に帰してしまう。ここでは、論文試験のどこが難しく感じる点であり、どのように対処すればよいのか考えてみたいと思う。

## 3.1.1 午後II試験は難しい試験？
論文試験が難しいと思われる理由は3つある。

(1) 時間が足りない
午後II試験では、標準で2,500～3,000字程度の文章を書くことになる。原稿用紙6ページ（以下、1ページ400字）を軽く超えた量である。しかも制限時間は2時間である。情報処理業界に従事する者にとって、このような量の文章を一定時間内に、手書きで書く機会はそうないはずである。受験者は時間に追われ、準備をしていなければ、目標の3分の1も書けないで試験時間が終わるとなるかもしれない。 この時間の少なさが、午後II試験の難しさの1つになっている。キーボードによる入力に慣れ、手書きで原稿を書いたことのない受験者にとって、高いハードルになっているのは間違いないだろう。

(2) どのような論点で書けばよいのか分からない
論文は、論述者が持っている主張を文字にして論じ、読み手を納得させようとするものである。しかし、いざペンを持って原稿用紙に向かってみると、何も思い浮かばない、何を書いてよいのか分からないといった状態になる。 論文と言われると、難しい言葉を駆使して、何か遠大なことを論じなければならないと考え込んでしまうからかもしれない。準備をしていない者にとって、論文という言葉自体の重みで、混乱し、論述する対象を見失う。これが論文試験を難しく感じさせる2つめの要因である。

(3) こんな経験をしたことがない
論文試験を難しく感じさせる3つめの要因は、「経験」である。論文試験では、毎回様々な題材が与えられる。例えば、組織再編、法改正対応、業務プロセスの根本的な見直し、大規模災害対応、AIの活用、グローバルな開発体制など様々な条件が、問題文や設問で指定される。受験者はいずれかの問題を選択し、論述しなければならない。しかしそれらの条件を全て経験している受験者は、おそらくいないはずである。つまり経験のないことに対して論述しなければならないことが多いのである。 「こんな、経験したことがない話題を論文にできるわけがないですよ！」そんな声をよく耳にする。準備していない受験者の率直な感想だと思う。このように「経験」していないことが題材になっていることが、受験者をして、論文試験を一層難しく感じさせる要因になっていると思われる。

## 3.1.2 どうすれば論文が書けるようになるか
論文が書けるようになるためには、前述の3つの要因に対処すればよい。

順を追って説明すると、3つの要因に対しては次のように対処すればよい。
「時間が足りない」という要因に対しては： 「論述をできるだけ単純作業化して、考え込まずに論述する」ということが必要である。これに関しては、論文の「設計書」を作成し、それに従って論述することで対応できる。設計書を作ってしまえば、あとは比較的短時間で論述できるのである。また、時間を測定しながら自分のペースを把握することも大切である。設計書と時間管理法に関しては、以降の節で詳しく解説する。

「どのような論点で書けばよいのか分からない」という要因に対しては： 「設問分解法」で対処する。一見すると、設問は抽象的なことを漠然と要求しているように思える。しかし、設問を分解すると、問題文と対応しており、具体的な論述内容に関する要求や指示が含まれていることが分かる。これに単純に「応えて」いけば、その蓄積が自動的に論文となるのである。この設問分解法に関しては、「3.3 論文設計書の作成（章、節の設計）」で詳述する。

「こんな経験をしたことがない」という要因に対しては： 「準備モジュール」で対処できる。準備モジュールとは、論述者が書き出しやすい話題やネタを、フレキシブルに論述に加工しやすいようにしたものである。本書ではひな型を用意したので、これに各人が書きやすい内容を追加して、未経験の話題でも柔軟に対処できるようにする。本書では「3.6 準備モジュールの適用」で詳しく解説する。

この他、「……することにした。なぜならば……」の繰り返し表現や、専門用語、準専門用語、定義表現、論文に使用する漢字などの基盤を身に付けることができれば、論文は誰にでも書けるようになる。以降で、これらの具体的な対策を解説していく。
