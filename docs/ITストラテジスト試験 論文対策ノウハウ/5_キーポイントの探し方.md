# 1. 論文執筆の基本：キーポイントの探し方
論文を合格レベルに引き上げるためには、問題文（問題冊子に記載された背景説明）の中に隠された「書くべき要素（キーポイント）」を正確に特定し、自分の論文に組み込む必要があります。

**探すべき2つのポイント**
- 設問に含まれている言葉を探す： 設問（ア、イ、ウ）で要求されている単語を問題文から探し出し、それに関連する具体的な内容を記述します。
- 箇条書きに注目する： 問題文の中にある箇条書きは、そのまま「論理構成の柱」として活用できる典型例です。

# 2. 年度別・設問別キーポイント抽出（Few-shot用サンプル）
過去問の分析から導き出された、論理構成のテンプレートとなる具体例です。

【令和5年：システムリスク対応方針】

| 設問 | 要求事項 | 論文に盛り込むべきキーポイント（例） |
| :--- | :--- | :--- |
| 設問ア | 収集した情報 | 現状の業務プロセス、ITシステムの機能、制約事項、前提条件 |
| | 分析方法 | 客観的に現状の分析を行う（全社視点での多面的分析） |
| 設問イ | 特定した問題の真因 | 改修要望が挙がってきた問題の真因を特定する |
| | 工夫したこと | 故障によるシステム停止、人為的ミス、サイバー攻撃による情報漏えい等のインシデント想定 |

【令和4年：新サービス立案】
[設問イ] 立案に当たり工夫した点
- 企画した新商品や新サービス： 「健康に気を使えば保険料を割り引く、健康増進型の保険商品を企画した」
- 顧客との接点や関係性： 「契約期間中の顧客との接点を増やし、顧客に新たな価値を感じてもらえる新商品を実現」
- 扱うデータ： 「健康診断や歩行などの健康データ」

【令和元年：デジタル技術の活用】
[設問ア] 活用したデジタル技術と実現した業務プロセス
- 音声認識装置とAIの活用： 「看護に直接関わらない業務を特定し、その中で記録業務プロセスに音声認識装置とAIの活用を図った」
- AR機器とIoTの活用： 「熟練作業者が行っていた組立業務プロセスにAR機器とIoTの活用を図った。これにより熟練作業者と同等の作業水準が達成され、事業課題を解決した」

# 3. 論理構成の構造化ルール合格論文を書くための「鉄則」として、以下のノウハウを構造化します。

## 3.1 真因の特定（問題解決の論理）
単なる「不具合」ではなく、なぜそれが起きたのかという **「真因（Root Cause）」** を特定するプロセスを記述します。
- Before: タブレット端末を導入して効率化した。
- After: 現状の分析（多面的分析）の結果、入力ミスという表面的な問題ではなく、情報共有のタイムラグが真因であることを特定し、リアルタイム連携を工夫した。

## 3.2 具体的な数値・固有名詞の導入
「効率化した」だけでは不十分です。以下の要素を必ず含めます。
- 対象範囲： どの部門の、どのプロセスか。
- 活用技術： AI、IoT、クラウド、SaaS、POS情報、QRコード決済など。
- 定量的効果： 導入コストの削減、売上の向上、作業時間の短縮。

# 4. 統合ガイド：AI論文生成用プロンプト（指示書）
以下の内容をシステムプロンプトや指示文に含めることで、精度の高い論文の下書きが生成できます。

ITストラテジスト午後II 論文生成ガイドライン
1. 構成の指示：
- 設問ア： 事業概要、業務特性、IT活用の背景、および「収集した情報の分析方法」を具体的に記述せよ。
- 設問イ： 課題の真因を特定し、解決策としてのIT戦略を記述せよ。その際、問題文の箇条書きにあるような「デジタル技術（AI/IoT/AR等）」の具体的活用方法を1つ以上含めること。
- 設問ウ： 評価と改善。設定した目標（KPI）に対する達成度を数値で評価し、今後の課題を述べよ。

2. 記述のトーン＆マナー：
- 「～と考える」「～を特定した」「～を図った」といった専門家（ストラテジスト）らしい決然とした口調で記述する。
- 抽象的な表現を避け、「10%のコスト削減」「特定の個人に偏らない意見収集」など具体性を持たせる。

3. 必須キーワードの埋め込み：
- 「経営戦略との整合性」「業務プロセスの変革」「利害関係者（ステークホルダ）との合意形成」「投資対効果（ROI）」の観点を必ず論理の中に組み込むこと。
