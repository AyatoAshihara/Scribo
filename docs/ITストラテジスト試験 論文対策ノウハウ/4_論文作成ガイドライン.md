# 4_論文作成ガイドライン
## 1. 論文全体の基本構成と執筆ルール
論文は、採点者が一読して論理の整合性を確認できるよう、以下のフレームワークで記述する。

### 1.1 論理展開の基本文型
すべての節・項の書き出しは、以下のパターンを繰り返すことで説得力を高める。

【基本パターン】
結論： 「私は、〇〇することにした。」
理由： 「なぜならば、〇〇だからである。」
行動： 「そこで私は、〇〇を実施した。」
結果： 「その結果、〇〇となった。」

### 1.2 「具体性」と「定量化」の原則
「具体的に述べよ」という設問指示に対し、数字と専門用語を織り交ぜてリアリティを出す。

定量化の例： | 抽象的な表現 | 定量的な記述例（合格レベル） | | :--- | :--- | | 生産性が向上した | 重複プロセスを統合し、生産性を30%アップさせた。 | | 短期間で回収する | 初期投資を3年以内で回収する計画を策定した。 | | 高い精度を実現した | AI連携OCRにより、90%以上の読み取り精度を確認した。 |

## 2. 第1章：事業概要と事業特性（設問ア対策）
自身の役割と、分析対象となる企業の背景を簡潔に述べる「背景モジュール」。

【執筆のポイント】
業種、事業内容、IT化の現状をセットで記述する。

利用部門からの要望だけでなく、ITストラテジストとしての「気づき」を含める。

【記述例（Few-shot）】
「A社は、海外の珍しい商品を多品目輸入する卸販売会社である。従来は営業担当者が直接出向くスタイルが主だったが、近年は通販サイトによる個人向け直接販売も開始した。今回の要望は営業部門からの『在庫情報の鮮度向上』であったが、私は一部門の視点に留まらず、全社的な経営戦略との整合性を確認することから着手した。」

## 3. 第2章：課題解決への取り組み（設問イ対策）
論文の「肝」となる部分。具体的な行動と、ITストラテジストとしての意思決定の根拠を示す。

【専門用語の活用例】
「コアコンピタンスの強化」「属人化の排除」「コスト体質の脱却」「ROIの最大化」

【論理展開の記述例（Few-shot）】
「私は、改善要望に対して営業部門以外の2部門（物流・情報システム）にもインタビューを行うことにした。なぜならば、 在庫情報の遅延は営業の入力漏れだけでなく、物流側の受発注プロセスの制約に起因していると考えたからである。そこで私は、 各部門の責任者を集めた個別面談を行い、現状の業務プロセスを可視化した。その結果、 ボトルネックとなっている手作業工程を特定し、クラウドを活用したリアルタイム共有システムの導入を提案するに至った。」

## 4. 第3章：評価・反省・今後の展望（設問ウ対策）
実施した施策に対する客観的な評価と、将来に向けた継続的な改善策を述べる。

【記述例：経営層からの評価（Few-shot）】
「施策の提案後、経営会議の席で、中期経営計画の目標値に照らし合わせた投資対効果（ROI）を説明した。経営層からは、『戦略との整合性が明確であり、良好な投資判断材料である』との一定の評価と承認を得ることができた。」

【記述例：今後の課題（Few-shot）】
「システム導入により情報の鮮度は向上したが、現場での入力負荷という新たな課題も認識している。今後は、スマートデバイスの音声入力を活用するなど、現場のUI/UXをさらに改善し、データの定着化を一段と進める計画である。」

## 5. 最終チェックと仕上げ（テクニカル・ルール）
執筆終了直前の12分で行うべき「合格の精度」を上げるための確認事項。

「止め」マークの確認： 指定された文字数（3ページ目中段など）を超過していないか、あるいは不足していないか。
章・節・項の整合性： タイトルが左詰めになっているか、インデントミスはないか。
一貫性チェック： 設問アで書いた課題が、設問ウで正しく評価・解決されているか。
