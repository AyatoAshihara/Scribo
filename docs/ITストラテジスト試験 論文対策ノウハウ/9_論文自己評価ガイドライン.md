# 1. 論文自己評価ガイドライン：基本ルール

試験本番や練習において、論述内容が形式要件を満たしているかを確認するための基本指針です。

## 執筆の物理的ルール（形式要件）
- **段落分けの徹底**: 意味の切れ目や話題が変わるタイミングで適切に改行し、一字下げ（インデント）を行う。
- **視認性の確保**: 原稿用紙のマス目に従い、読みやすい文字で記述する。
- **文字数制限**: 指定された文字数（下限・上限）を厳守する。

| 項目 | 不適切な例（×） | 適切な例（○） |
| :--- | :--- | :--- |
| 段落分け | 切れ目がなく、見栄えが悪い。どこまでが一段落か判別しにくい。 | 項目（2-1など）の書き出しや内容の転換点で適切に改行されている。 |
| 可読性 | 話題の転換が唐突。読み手に負担をかける。 | 「なぜならば」「具体的には」「そして」等の接続詞と段落で見通しが良い。 |

# 2. 論文評価チェックシート（減点ポイントの構造化）

自己採点やAI添削時に使用する、5段階の減点レベルと評価項目です。

### レベル1：不合格（-41点～）
- **テーマの乖離**: 論述内容がテーマと大きく異なっている。
- **構成の不備**: 章タイトルや節タイトルがない、論述していない章がある。
- **設問無視**: 設問の要求事項に明らかに答えていない、無視している。
- **資格不適合**: 資格区分（ITストラテジスト等）にふさわしい観点でない。
- **用語の誤用**: 専門用語を誤用している。
- **態度の不備**: 建設的でない、批判的である。

### レベル2：大減点（-20点～）
- **論理の一貫性**: 選択した問題と論述内容が異なる、章・節タイトルに誤りや不足がある。
- **要求事項の漏れ**: 明確に答えていない要求事項がある。
- **舞台設定の不備**: どのような企業を舞台にしているか不明瞭。
- **専門性の欠如**: 資格区分の知識がやや不足している。
- **具体性の欠如**: 具体性が不足している、客観性が不足している、一般論に終始している。

### レベル3：中減点（-13点～）
- **文章構成**: 一文が長過ぎて意味を読み取りにくい箇所がある。
- **不明瞭な表現**: 意味が分からない箇所がある、特殊な業界用語・社内用語があり説明もない。
- **日本語の誤り**: 「てにをは」がおかしい、誤字・脱字がある。

### レベル4：小減点（-10点～）
- **形式の乱れ**: 記述がセンタリングされている、インデントの量が多すぎる。
- **マスの使い方**: 原稿用紙のマス目から文字がはみ出している。
- **文脈の乱れ**: 主語と述語が不明確、句読点の位置がおかしい。

### レベル5：微減点（-7点～）
- **表記のゆれ**: 特殊文字を多用している。
- **具体性の微細な欠如**: 定量的表現がまったくない、専門用語が少ない。

# 4. 統合ガイド：AI用「論文生成・添削」プロンプト

この資料のノウハウをAIに指示するためのプロンプト形式です。

```markdown
【AIへの指示書：ITストラテジスト論文生成】
# 役割
あなたはITストラテジスト試験の合格論文を執筆するエキスパートです。

# 執筆の基本原則
- **資格区分の視点**: 常に「経営層の視点」を持ち、ITを活用した事業変革を立案・主導する立場を維持してください。
- **具体性と定量性**: 一般論を避け、業界背景、企業規模、具体的な数値目標を含めてください。
- **設問への忠実な回答**: 設問ア、イ、ウの各要求事項に対し、漏れなく回答してください。

# 構造と形式
- **章・節タイトル**: 必ず章（第1章）と節（1-1）を明記すること。
- **段落**: 意味のまとまりごとに改行し、先頭は一字下げすること。
- **文体**: 「私は……」という一人称を用い、断定的かつ建設的な表現（だ・である調）にすること。

# 評価基準（減点回避）
以下の項目に該当しないよう注意してください：
- 専門用語の誤用、一般論への終始。
- 批判的な態度や、技術的な詳細（開発者目線）への偏り。
- 主語・述語の不一致や、一文が長すぎる記述。
```
