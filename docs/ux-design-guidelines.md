# Scribo UXデザインガイドライン

本ドキュメントは、IPA午後Ⅱ論述式試験の学習支援アプリ「Scribo」のUX/UIデザインにおける指針を定める。

## 基本原則

### 1. 認知負荷の最小化 (Cognitive Load)
ユーザーが情報を処理・理解する際の精神的エネルギーを最小限に抑える。

**適用例:**
- 問題文と回答欄を左右分割レイアウトで同時表示
- 設問ごとにタブで区切り、一度に一つの設問に集中
- 文字数カウントをリアルタイム表示し、計算の手間を省く

### 2. ドハティの閾値 (Doherty Threshold) - 0.4秒の壁
ユーザーが0.4秒以上待たされると興味喪失のリスクが高まる。

**適用例:**
- ページ遷移時は即座にスケルトンローダーを表示
- API応答待ちにはスピナーを0.1秒以内に表示
- 採点処理（90秒程度）は「労働の錯覚」と組み合わせる

### 3. 労働の錯覚 (Labor Illusion)
処理に時間がかかる場合、労力を可視化することで価値を感じさせる。

**適用例:**
- 採点中に「8観点で分析中...」「文章構成を評価中...」などの段階表示
- プログレスバーで進捗を可視化
- 「AIが丁寧に採点しています」などのメッセージ

### 4. 目標勾配効果 (Goal Gradient Effect)
目標に近づくにつれ、行動や努力を加速させる。

**適用例:**
- 回答の文字数が目標に近づくとバッジの色を変える（警告→成功）
- 全設問の回答状況をインジケーターで表示
- 「あと○文字で目標達成」のメッセージ

### 5. ピーク・エンドの法則 (Peak-End Rule)
体験の評価は、最高の瞬間と終わりの瞬間で決まる。

**適用例:**
- 採点結果は視覚的に印象的なデザインで表示（ランクA/B/C/D）
- 合格時は祝福アニメーション
- 学習セッション終了時にポジティブなフィードバック

## インタラクションパターン

### 6. 段階的開示 (Progressive Disclosure)
情報や機能を段階的に表示し、必要な時に必要な情報にアクセスさせる。

**適用例:**
- 採点結果は最初にランクのみ表示、詳細は展開可能に
- 問題一覧は基本情報のみ、詳細はホバー/クリックで
- 初回利用時のチュートリアルは段階的に

### 7. デフォルト効果 (Default Bias)
適切なデフォルト値を設定し、ユーザーの意思決定を支援。

**適用例:**
- タイマーのデフォルト値は試験時間（120分）
- 最初の設問タブをデフォルトで選択
- 自動保存をデフォルトで有効

### 8. 損失回避 (Loss Aversion)
潜在的な損失を避ける傾向を活用。

**適用例:**
- 回答未保存時の離脱警告
- 「入力内容が失われます」の確認ダイアログ
- ローカルストレージへの自動保存で安心感

### 9. フレーミング効果 (Framing)
情報の提示方法で印象が変わる。

**適用例:**
- 「残り時間: 30分」より「経過時間: 90分 / 120分」
- 「不合格」より「あと○点で合格ライン」
- エラーメッセージは解決策とセットで

## ビジュアルデザイン

### 10. 視覚的階層 (Visual Hierarchy)
視覚的に優先順位を付けて情報を整理。

**適用例:**
- 問題タイトル > 本文 > 設問 の優先度で文字サイズ設定
- CTAボタン（採点送信）は最も目立つ色
- 現在選択中のタブは明確に強調

### 11. 美的ユーザビリティ効果 (Aesthetic-Usability Effect)
美しいデザインは使いやすく感じられる。

**適用例:**
- DaisyUIのテーマで一貫性のあるデザイン
- 適切な余白（padding/margin）で読みやすさ確保
- アニメーションは控えめだが滑らかに

### 12. 親近性バイアス (Familiarity Bias)
既知のデザインパターンを活用。

**適用例:**
- 一般的なアイコン使用（⏱️タイマー、💾保存、📝編集）
- フォームのラベル配置は標準的に
- ナビゲーションは左サイドバー

## フィードバック

### 13. 社会的証明 (Social Proof)
他者の行動が信頼性を高める。

**将来の適用例:**
- 「○人がこの問題に挑戦しました」
- 平均スコアとの比較
- 人気の問題ランキング

### 14. ユーザー歓喜効果 (User Delight)
期待を超える喜びの瞬間を提供。

**適用例:**
- 合格時の祝福エフェクト（紙吹雪など）
- 連続学習日数の表示
- ちょっとした励ましメッセージ

## エラー処理

### 15. 意図的な壁 (Intentional Friction)
重要な操作には確認ステップを設ける。

**適用例:**
- 回答送信前の確認ダイアログ
- 文字数不足時の警告
- タイマー終了時の選択肢提示

## コンポーネント設計方針

### ボタン
- プライマリ: 採点送信など主要アクション → `btn-primary`
- セカンダリ: キャンセル、戻るなど → `btn-ghost`
- 危険: 削除、リセットなど → `btn-error`

### カラーコーディング
- 成功/合格: `badge-success`, `text-success`
- 警告/注意: `badge-warning`, `text-warning`
- エラー/不合格: `badge-error`, `text-error`
- 情報: `badge-info`, `text-info`

### ローディング状態
```html
<!-- インライン -->
<span class="loading loading-spinner loading-sm"></span>

<!-- フルスクリーン -->
<div class="flex justify-center items-center h-full">
    <span class="loading loading-spinner loading-lg text-primary"></span>
</div>

<!-- スケルトン -->
<div class="skeleton h-4 w-full"></div>
```

### アラート
```html
<!-- 成功 -->
<div class="alert alert-success">
    <span>採点が完了しました</span>
</div>

<!-- エラー -->
<div class="alert alert-error">
    <span>エラーが発生しました</span>
    <button class="btn btn-sm">再試行</button>
</div>
```
